Inheritance

Inheritance can be defined as the process where one class acquires the properties (methods and fields) of another. With the use of inheritance the information is made manageable in a hierarchical order. The class which inherits the properties of other is known as subclass (derived class, child class) and the class whose properties are inherited is known as superclass (base class, parent class).

The idea of inheritance is simple but powerful: When you want to create a new class and there is already a class that includes some of the code that you want, you can derive your new class from the existing class. In doing this, you can reuse the fields and methods of the existing class without having to write (and debug!) them yourself.

A subclass inherits all the members (fields, methods, and nested classes) from its superclass. A subclass inherits all of the public and protected members of its parent. You can use the inherited members as is, replace them, hide them, or supplement them with new members.

Step 1: Extending a Class

- We will now extend the Fraction class to create mixed fraction. For example 2(3/4).
- The 'numerator' and 'denominator' defined in Fraction class cannot be accessed from 
  MixedFraction since they are 'private'. Try and check. You will remember.
- To access them, the access specifier needs to be changed from 'private' to 'protected'.
  A 'protected' member can be accessed from both the class and the subclass.
  So, first change them and recompile Fraction class.
- A MixedFraction will now derive 'numerator' and 'denominator' from Fraction class. 
  In addition it also adds a new attribute 'wholeNum'.
- The MixedFraction, with getter and setter, can be defined as follows. 
  Note the 'extends' keyword.
  public class MixedFraction extends Fraction {
      private int wholeNum;
      public void setWholeNum(int w) {
        wholeNum = w;
      }
      public int getWholeNum() {
        return wholeNum;
      }
  }
- The driver class can now instantiate, set and get the mixed fraction.
      MixedFraction mf = new MixedFraction();
      mf.setWholeNum(2);     // Calling method from this class
      mf.setNumerator(3);    // Calling method from parent class
      mf.setDenominator(4);  //         --do--
      System.out.println(mf.getWholeNum() + "(" 
                          + mf.getNumerator() 
                          + mf.getDenominator() + ")");
- Calling a superclass methods can be done in the same fashion as if they were part of
  subclass definition. You can notice mf.getNumerator() in the above code.

Step 2: Overriding Principle

- You can also add a print method in MixedFraction to print the mixed fraction.
    public void print() {
      System.out.println(wholeNum + "("
                          + numerator 
                          + denominator + ")");
    }
  The last statement in driver can be replaced by mf.print().
- The Fraction class already has a print() method with the same signature. 
  What happens to it? It is overridden.
- The print() method in MixedFraction overrides the print() method in Fraction.
- When you implement a new method in the subclass that has the same signature as the one 
  in the superclass, then the subclass method is said to override the super class method.
- In a similar fashion, the inherited fields can also be used directly from the subclass.
  In the above code for print(), you can notice mf.numerator and mf.denominator.
- You can declare a field in the subclass with the same name as the one in the superclass, 
  thus hiding it (not recommended). Try it and see for yourself.

Step 3: Calling overridden super class method from subclass

- The print() method can be re-written in such a way that it calls the print() method
  of the Fraction class too.
    public void print() {
      System.out.print(mf.wholeNum + "("); 
      super.print(); 
      System.out.println(")"); 
    }
- This prints the whole number first, followed by a call to super class's print method
  and finally closes the bracket.
- You can write a new static method in the subclass that has the same signature as 
  the one in the superclass, thus hiding it.
- Try and understand the netigrities properly.

Step 4: Working with Constructors

- Constructors are not inherited by subclasses, but the constructor of the superclass 
  can be invoked from the subclass.
- Below is the code for explicit invocation from subclass constructor
     MixedFraction(int w, int n, int d) {
       super(n,d);
       wholeNum = w;
     }
- In the above case, Fraction's construction is explicitly invoked
- You can check its working by writing the following driver code
      MixedFraction mf = new MixedFraction(2,3,4);
      mf.print();
  This should output 2(3/4)
- Implement default constructor

Step 5: Casting Objects

- All mixed fractions are fractions. So you can assign as follows.
    Fraction f = new MixedFraction(); // meaning every mixed fraction is a fraction
- But the reverse is not possible
    MixedFraction mf = new Fraction(); // meaning all fractions are not mixed fractions
- This applies to assigning return values too. If a method returns a MixedFraction, it
  can be casted and assigned to Fraction object.
     Fraction f = (Fraction) mf1.add(m2);  // This is legal. mf1 & mf2 are mixed fractions
- But the following is not legal
     MixedFraction mf = (MixedFraction) f1.add(f2);  // Illegal. f1 & f2 are fractions

Step 6: Implement all the methods

- Implement all the methods like inverse, reduce, equals, add, etc.
- Note that for most of the methods, it makes sense to first convert to fraction form,
  call the concern superclass method, get the result and re-convert back to mixed fraction.
- It is better to have 2 additional methods which does this forward and reverse conversion
  in MixedFraction and Fraction
     public Fraction toFraction() { // To be implemented in MixedFraction
          // Your code here
     }
     public MixedFraction toMixedFraction() { // To be implemented in Fraction
          // Your code here
     }
- Now add method in MixedFraction can be implemented as follows:
      public MixedFraction add(MixedFraction mFrac) {
         Fraction f1 = toFraction();  // conversion of this Fraction object
         Fraction f2 = mFrac.toFraction(); // conversion of mFrac object
         Fraction sum = f1.add(f2);  // sum of the two fractions - reusing the code
         return sum.toMixedFraction();  // reconvert back to MixedFraction
      }
- Implement rest of the methods
- Run javadoc to generate documentation of all the classes

A Note on Multiple Inheritance

- Extending multiple classes simultaneously is not possible in Java. For example
  if A and B are two classes, we cannot do as follows:
     public class C extends A, B {
              :
     }
- This is a C++ feature but prone for misuse and leads to all kinds of problems
  which are difficult to debug.
- So Java has avoided this feature. You can extend only one class at a time.
